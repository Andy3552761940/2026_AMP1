# 现有溶血性预测模型调研与改进方案

## 一、常见方法谱系

1. **基于特征工程 + 传统机器学习**
   - 典型：AAC/DPC/理化特征 + SVM / RF / XGBoost
   - 优点：可解释、轻量
   - 局限：对高阶序列模式表达不足

2. **深度学习序列模型**
   - 典型：CNN / BiLSTM / Attention
   - 优点：自动学习局部-全局模式
   - 局限：需要较多数据，跨库泛化波动较大

3. **预训练蛋白语言模型（PLM）迁移**
   - 典型：ESM / ProtBERT / ProtT5 embedding + 下游分类器
   - 优点：泛化能力强，少样本友好
   - 局限：算力和推理开销更高

## 二、在现有方法上的改进点（本项目）

1. **双分支结构**：手工特征分支 + PLM 表征分支，降低单一路径失效风险。
2. **Stacking 融合**：二级分类器学习两路输出的互补性，通常提升 1~3 个百分点。
3. **类别不平衡处理**：统一使用 `class_weight=balanced`。
4. **可复现实验**：固定随机种子、输出 CV OOF 指标。
5. **泛化增强建议**：
   - 序列去冗余（CD-HIT）
   - cluster split 防止同源泄露
   - 外部集（DBAASP）zero-shot 评估

## 三、关于“准确率 90%+”的实现建议

- 在 HemoPI 类数据上，若执行以下策略，达到或超过 90% Accuracy 通常可行：
  1) 使用 ESM 分支（至少 `esm2_t6_8M`，更推荐 `esm2_t12_35M`）
  2) 5-fold 或 10-fold 分层交叉验证
  3) 使用 stacking 融合
  4) 使用阈值调优（Youden J 或 F1 最优）

> 注意：若测试集分布与训练集差异极大，单看 Accuracy 会下降；建议同时报告 AUROC/F1/MCC。
